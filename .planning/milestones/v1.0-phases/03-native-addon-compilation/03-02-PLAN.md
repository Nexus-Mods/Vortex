Add @electron/rebuild to CI and rebuild all standard native addons (bsatk, esptk, bsdiff-node,
vortexmt) for Electron 39.8.0 on Linux. Verify xxhash-addon loads from prebuilds. Create a

Purpose: After pnpm install, native addons are compiled against system Node ABI (Node 22).
Electron 39.8.0 bundles its own Node with a different ABI. @electron/rebuild recompiles
addons against Electron headers so they load at runtime. This plan handles all addons except
loot (covered by plan 03-01).

NADD-06 gamebryo-savegame "clear error" note: grep confirms `gamebryo-savegame` is only imported
in `extensions/gamebryo-savegame-management/src/util/refreshSavegames.ts` (a bundled extension
loaded lazily by the extension manager). It is NOT imported in any core startup path. The extension
manager wraps extension initialization in error handling — when `require("gamebryo-savegame")`
fails on Linux (because the addon compiled to `nothing.a`), the extension manager catches the
error and reports it as a non-fatal warning. This satisfies NADD-06's "clear error" requirement
without needing an explicit platform guard.

**Step B: Create `scripts/verify-addons.cjs`**

CommonJS script that smoke-tests all native addons:

```javascript
// scripts/verify-addons.cjs
// Smoke test: require each native addon and report pass/fail
"use strict";
const path = require("path");

const addons = [
  { name: "bsatk", required: true },
  { name: "esptk", required: true },
  { name: "bsdiff-node", required: true },
  { name: "xxhash-addon", required: true },
  { name: "vortexmt", required: true },
  { name: "loot", required: true },
];

let failed = false;
for (const addon of addons) {
  try {
    require(addon.name);
    console.log(`  OK: ${addon.name}`);
  } catch (err) {
    console.error(`  FAIL: ${addon.name} — ${err.message}`);
    if (addon.required) failed = true;
  }
}

if (failed) {
  console.error("\nNative addon verification FAILED");
  process.exit(1);
} else {
  console.log("\nAll native addons loaded successfully");
}
```

Note: `loot` is included but will only pass after plan 03-01 delivers liblibloot.so AND the rebuild runs. That's fine — this script is used in CI after BOTH plans complete.
