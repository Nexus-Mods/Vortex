<objective>
Build libloot 0.29.1 from source for Linux and deliver it as liblibloot.so so the loot native addon can compile.

Purpose: LOOT stopped publishing Linux prebuilts at v0.24.5. The node-loot package pins v0.29.1.
Without libloot.so, loot.node produces a `nothing.a` stub on Linux. This plan builds libloot
from source using cmake + cargo (Rust), delivers the .so via a postinstall script, and adds the
necessary CI infrastructure (Rust toolchain, cmake, apt deps).

Output: `scripts/postinstall-libloot.cjs`, updated `.github/workflows/main.yml`, updated `package.json`

NOTE: D-02 specified sourcing from LOOT GitHub release prebuilts, but research confirmed prebuilts
do not exist for v0.29.1. Building from source is the only viable path. D-01 (add libloot.so to
node-loot) and D-03 (postinstall delivery) are honored as specified.
</objective>

Create `scripts/postinstall-libloot.cjs` (CommonJS — must work with bare `node` in postinstall context):

1. Guard: if `process.platform !== 'linux'`, log "postinstall-libloot: skipping (not Linux)" and `process.exit(0)`.

2. Locate loot_api dir dynamically:
   - Use `require.resolve('loot/package.json')` wrapped in try/catch. If loot is not installed (e.g., optional dep), exit 0 gracefully.
   - `const lootApiDir = path.join(path.dirname(lootPkgPath), 'loot_api');`

3. Check if `liblibloot.so` already exists at `path.join(lootApiDir, 'liblibloot.so')`. If yes, log "liblibloot.so already present, skipping build" and exit 0.

4. Build libloot 0.29.1 from source:
   ```
   const LIBLOOT_VERSION = '0.29.1';
   const tmpDir = path.join(os.tmpdir(), 'libloot-build');
   ```
   - Run: `git clone --depth=1 --branch ${LIBLOOT_VERSION} https://github.com/loot/libloot.git ${tmpDir}`
   - Run: `cmake -DCMAKE_BUILD_TYPE=Release -DLIBLOOT_BUILD_TESTS=OFF -DLIBLOOT_INSTALL_DOCS=OFF -B ${tmpDir}/cpp/build -S ${tmpDir}/cpp`
   - Run: `cmake --build ${tmpDir}/cpp/build --parallel`
   - Use `child_process.execSync` with `{ stdio: 'inherit' }` for all commands.
   - Before cmake, source cargo env: check if `$HOME/.cargo/env` exists and prepend `source $HOME/.cargo/env &&` to cmake commands, OR just rely on cargo being on PATH (CI installs it before pnpm install).

5. Copy the built `.so`:
   - With `-DCMAKE_BUILD_TYPE=Release` and the default Unix Makefiles generator on Linux, cmake
     outputs the library directly at `${tmpDir}/cpp/build/liblibloot.so` (no Release/ or
     RelWithDebInfo/ subdirectory). Use this single deterministic path.
   - Copy to `path.join(lootApiDir, 'liblibloot.so')`.
   - The linker flag `-l../loot_api/libloot` causes the linker to search for `liblibloot.so`
     (standard -l behavior prepends `lib` and appends `.so`). The cmake output is already named
     `liblibloot.so`, so copy as-is.

6. Clean up: `fs.rmSync(tmpDir, { recursive: true, force: true })`.

7. Wrap entire build in try/catch. On failure: log the error, log "WARNING: libloot.so build failed — loot addon will not be available on Linux", and `process.exit(0)` (non-fatal — don't block pnpm install for other platforms or optional features).

Wire into package.json:
- If no existing `postinstall` script: add `"postinstall": "node scripts/postinstall-libloot.cjs"`
- If existing `postinstall` script: chain with `&&` — `"postinstall": "existing-command && node scripts/postinstall-libloot.cjs"`

```yaml
- name: Install Rust toolchain

  uses: dtolnay/rust-toolchain@stable

- name: Install native addon build dependencies

```

Notes:
- `dtolnay/rust-toolchain@stable` is the standard GH Action for Rust. It puts `cargo` and `rustc` on PATH.
- `cmake` is needed by the postinstall-libloot.cjs script (called during `pnpm install`).
- `liblz4-dev` is for potential gamebryo-savegame compilation (plan 03-03 decides).
- Do NOT add a separate apt-get update — the existing "Install fontconfig" step already runs `apt-get update`. Just add `apt-get install -y` without update (apt cache is still warm from the prior step). Actually, to be safe since steps may be cached independently, keep `sudo apt-get update &&` in the new step OR merge with the existing fontconfig step. Merging is cleaner — update the existing "Install fontconfig" step to also install cmake and liblz4-dev:

```yaml
- name: Install Linux build dependencies
  if: runner.os == 'Linux'
  run: sudo apt-get update && sudo apt-get install -y libfontconfig1-dev cmake liblz4-dev
```

This replaces the existing "Install fontconfig" step (rename + add packages).

**Ordering matters:** Rust toolchain and cmake MUST be available before `pnpm install` because the postinstall-libloot.cjs script runs during install and needs both `cargo` and `cmake` on PATH.

The "Install dependencies" (`pnpm install`) step will now trigger postinstall-libloot.cjs which builds libloot.so. No separate CI step needed for the libloot build itself.
