<objective>
Create the winapi-bindings Linux shim module with full export coverage using TDD.

Purpose: This shim replaces the Windows-only `winapi-bindings` native module on Linux. It provides functional implementations of `GetDiskFreeSpaceEx` and `GetVolumePathName` using Node.js built-ins, a throwing stub for `ShellExecuteEx`, and safe no-op/null stubs for all remaining exports. The shim must satisfy both default import (`import winapi from`) and namespace import (`import * as winapi from`) patterns.

Output: `src/renderer/src/util/winapi-shim.ts` (shim) and `src/renderer/src/util/__tests__/winapi-shim.test.ts` (tests)

    Implement each function category:

    **Functional stubs (WAPI-02, WAPI-03):**

    `GetDiskFreeSpaceEx(filePath: string): { total: number; free: number; freeToCaller: number }` — call `fs.statfsSync(filePath)`, return `{ total: stats.blocks * stats.bsize, free: stats.bfree * stats.bsize, freeToCaller: stats.bavail * stats.bsize }`. Let ENOENT propagate (callers catch per Pitfall 2 in RESEARCH.md).

    `GetVolumePathName(filePath: string): string` — `try { const targetDev = fs.statSync(filePath).dev; let current = path.resolve(filePath); while (true) { const parent = path.dirname(current); if (parent === current) return current; try { if (fs.statSync(parent).dev !== targetDev) return current; } catch { return current; } current = parent; } } catch { return path.parse(path.resolve(filePath)).root; }` — outer catch handles ENOENT on initial statSync by falling back to `path.parse(p).root` (always "/" on Linux per Pitfall 1).

    `GetNativeArch(): { nativeMachineCode: number; nativeArch: string; usedFallback: boolean }` — return `{ nativeMachineCode: 0, nativeArch: process.arch, usedFallback: false }`.

    `SetProcessPreferredUILanguages(_languages: string[]): void` — no-op, empty body.
