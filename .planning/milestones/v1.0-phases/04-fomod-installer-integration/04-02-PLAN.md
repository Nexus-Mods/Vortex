Present the following verification steps to the user. This is a human-verify checkpoint — no automated code changes.

What was built: Plan 01 added asarUnpack entries for Linux FOMOD binaries and fixed VortexIPCConnection to resolve ModInstallerIPC (no .exe) on Linux. Task 1 of this plan confirmed binaries exist on disk. Now we need to verify the full FOMOD installation flow works end-to-end on Linux.

Steps for the user to verify:
1. Run `pnpm run start` to launch Vortex on Linux
2. Add or select a game that supports FOMOD installation (any Bethesda game, e.g., Skyrim SE)
3. Download or manually add a mod that uses a FOMOD installer (any mod with a `fomod/ModuleConfig.xml` file)
4. Attempt to install the mod — Vortex should:
   a. Find and launch `ModInstallerIPC` (check console for "Found executable" log, NOT "Failed to find executable")
   b. The first two connection strategies (Sandbox + NamedPipe) will fail with warnings — this is expected on Linux
   c. The third strategy (TCP) should succeed — check for "Connected" or similar log
   d. `TestSupported` should return `supported: true` for the FOMOD mod
   e. The FOMOD installer dialog should appear (step selection UI)
5. If the FOMOD dialog appears and you can select options and complete the install: FOMD-04 is satisfied

Expected console output pattern:
- Warning: Sandbox launcher not supported on this platform (expected)
- Warning: Named pipe connection failed (expected)
- Info: TCP connection established (success)
- TestSupported result: supported=true

If it fails, check console for the specific error. Common issues:
- "Failed to find executable" — VortexIPCConnection.getExecutablePaths not returning correct path
- "Connection refused" — ModInstallerIPC failed to start (check if binary is executable)
- ".NET not found" — dotnetprobe could not detect .NET 9 runtime (install dotnet-runtime-9.0)

Resume signal: Type "approved" if FOMOD install completes successfully, or describe the error you see.
